# @fuggetlenfe/angular-wrapper

Auto-generated Angular bridge layer for the Stencil web components. This package contains zero component logic and zero styling. It provides Angular standalone directives that delegate everything to the underlying web component.

## Architecture role

```
packages/components          Stencil web components (logic + DOM contract)
      |
      |  stencil.config.ts → angularOutputTarget()
      v
packages/angular-wrapper     Auto-generated Angular directive proxies
  src/lib/generated/
    components.ts             @ProxyCmp decorated standalone components
    angular-component-lib/
      utils.ts                proxyInputs / proxyOutputs / ProxyCmp decorator
    index.ts                  DIRECTIVES array export
  src/index.ts                Public re-export
```

The wrapper is generated by `@stencil/angular-output-target` during the components build. Each generated directive uses `@ProxyCmp` to proxy Angular template bindings to the underlying web component element properties.

## What the wrapper does

- Registers the custom element (`defineCustomElement`)
- Maps Angular `@Input()` bindings to web component properties via `proxyInputs()`
- Maps web component events to RxJS observables via `proxyOutputs()`
- Uses `OnPush` change detection and `c.detach()` for optimal Angular performance
- Runs property writes outside Angular's zone (`runOutsideAngular`) to avoid unnecessary change detection

## What the wrapper does NOT do

- No styling
- No theme or brand logic
- No business logic
- No module dependencies (standalone components only)

## Usage

```typescript
// 1. Import the wrapper directive (logic bridge only)
import { FfButton } from '@fuggetlenfe/angular-wrapper';

@Component({
  imports: [FfButton],  // Standalone directive, no module needed
  template: `
    <!-- 4. Set brand and theme on the shell, use the directive in templates -->
    <main data-brand="brand-1" [attr.data-theme]="activeTheme()">
      <ff-button>Click me</ff-button>
      <ff-button [disabled]="true">Disabled</ff-button>
      <ff-button [fullWidth]="true">Full width</ff-button>
      <ff-button type="submit">Submit</ff-button>
    </main>
  `
})
export class AppComponent { }
```

```css
/* 2. Import the token contract in styles.css (stable CSS custom property API) */
@import '@fuggetlenfe/tokens/contract.css';

/* 3. Import a brand pack (visual identity) */
@import '@fuggetlenfe/brand-styles/brand-1-light.css';
@import '@fuggetlenfe/brand-styles/brand-1-dark.css';
```

### Per-instance token overrides

```html
<ff-button
  [style.--ff-button-radius]="'20px'"
  [style.--ff-button-padding-inline]="'1.3rem'">
  Custom tokens
</ff-button>
```

## Exported directives

| Export | Selector | Inputs | Description |
|---|---|---|---|
| `FfButton` | `ff-button` | `disabled`, `fullWidth`, `label`, `type` | Button primitive directive |
| `DIRECTIVES` | - | - | Array of all directive exports for convenience |

## Commands

```bash
pnpm --filter @fuggetlenfe/angular-wrapper build
```

The build depends on `@fuggetlenfe/components` being built first.
