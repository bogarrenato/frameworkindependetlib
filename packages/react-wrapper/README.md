# @fuggetlenfe/react-wrapper

Auto-generated React bridge layer for the Stencil web components. This package contains zero component logic and zero styling. It exists solely to provide React-compatible JSX exports that delegate everything to the underlying web component.

## Architecture role

```
packages/components          Stencil web components (logic + DOM contract)
      |
      |  stencil.config.ts → reactOutputTarget()
      v
packages/react-wrapper       Auto-generated React component wrappers
  src/generated/components.ts   createComponent() bridge for each web component
  src/index.ts                  Public re-export
```

The wrapper is generated by `@stencil/react-output-target` during the components build. The generated code uses `createComponent()` from `@stencil/react-output-target/runtime` to map web component properties and events to React props.

## What the wrapper does

- Registers the custom element (`defineCustomElement`)
- Bridges Stencil props to React props
- Bridges web component events to React event handlers
- Provides TypeScript types for all component props

## What the wrapper does NOT do

- No styling
- No theme or brand logic
- No business logic
- No state management

## Usage

```tsx
// 1. Import the wrapper (logic bridge only)
import { FfButton } from '@fuggetlenfe/react-wrapper';

// 2. Import the token contract (stable CSS custom property API)
import '@fuggetlenfe/tokens/contract.css';

// 3. Import a brand pack (visual identity)
import '@fuggetlenfe/brand-styles/brand-2-light.css';
import '@fuggetlenfe/brand-styles/brand-2-dark.css';
```

```tsx
// 4. Set brand and theme on the shell, render the wrapper component
<main data-brand="brand-2" data-theme="light">
  <FfButton>Click me</FfButton>
  <FfButton disabled>Disabled</FfButton>
  <FfButton fullWidth>Full width</FfButton>
  <FfButton type="submit">Submit</FfButton>
</main>
```

### Per-instance token overrides

```tsx
type CssTokenOverrides = CSSProperties & Record<`--ff-${string}`, string>;

const customStyle: CssTokenOverrides = {
  '--ff-button-radius': '20px',
  '--ff-button-padding-inline': '1.3rem',
};

<FfButton style={customStyle}>Custom tokens</FfButton>
```

## Exported components

| Export | Web component | Description |
|---|---|---|
| `FfButton` | `<ff-button>` | Button primitive with disabled, type, fullWidth, label props |
| `FfDropdown` | `<ff-dropdown>` | Single-select combobox with controlled `open` / `value` and JSON or array options |
| `FfDataTable` | `<ff-data-table>` | Sortable, selectable table with controlled sort and selection state |
| `FfModal` | `<ff-modal>` | Focus-trapped dialog with scroll lock and `ffClose` event |

## Server entry point (SSR)

The package exposes a dedicated server subpath, `@fuggetlenfe/react-wrapper/server`, which re-exports the Node-runnable hydrate API from `@fuggetlenfe/components/hydrate`. This is the React-side entry for Next.js, Remix, Astro, or any custom React SSR / SSG pipeline that needs to serialize `<ff-*>` elements into **Declarative Shadow DOM** at build time or per request.

```ts
import { renderToString } from '@fuggetlenfe/react-wrapper/server';

const html = `
  <main data-brand="brand-2" data-theme="light">
    <ff-button>Hello</ff-button>
  </main>
`;

const { html: rendered } = await renderToString(html, {
  serializeShadowRoot: 'declarative-shadow-dom',
  fullDocument: false,
});
```

The output contains `<template shadowrootmode="open">` blocks for every `<ff-*>` element and hydrates without a flash of unstyled content on Chrome 111+, Safari 16.4+, and Firefox 123+. The `hydrateDocument` counterpart is also re-exported for full-document rendering.

## Commands

```bash
pnpm --filter @fuggetlenfe/react-wrapper build
```

The build depends on `@fuggetlenfe/components` being built first.
