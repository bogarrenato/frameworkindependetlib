React Wrapper

Framework bridge for shared web components.

The React wrapper deliberately stays thin. It exposes React-friendly component APIs while keeping design, brand selection, and business logic outside the package.

UI kit reference

Use the same primitive logic in React without inheriting design decisions.

@fuggetlenfe/react-wrapper turns shared Stencil primitives into React-ready exports. The package does not own light and dark mode, brand activation, layout policy, or product-specific behavior. Consumers import wrapper components, then separately bring in the token contract and the official brand pack they want to render.

Getting started

A React consumer imports the wrapper package for component behavior and separately imports the token contract plus the official light and dark brand stylesheets it wants to use.

pnpm add @fuggetlenfe/react-wrapper @fuggetlenfe/tokens @fuggetlenfe/brand-styles
import '@fuggetlenfe/tokens/contract.css'; import '@fuggetlenfe/brand-styles/brand-2-light.css'; import '@fuggetlenfe/brand-styles/brand-2-dark.css'; import { FfButton, FfDropdown } from '@fuggetlenfe/react-wrapper';

Architecture boundaries

The React package is a bridge layer, not a second design system implementation. The component behavior stays in the shared web component core and the design remains in external token and brand packages.

What belongs here

React-friendly exports and JSX bridge behavior.
Type-safe event and property mapping for the shared primitives.
Stable consumption surface for React product teams.

What does not belong here

Brand switching logic or theme ownership.
Business-specific layouts or product state.
Hardcoded visual design tokens.

Component reference

The current wrapper documents the same public API that the shared Stencil primitives expose. Styling examples below assume the token contract and a brand pack are already loaded.

Primitive Stable PoC

FfButton

Logic-only button primitive for primary actions, secondary actions, and shell-level theme demonstrations. Visual identity is fully controlled by external tokens.

export function SaveAction() { return ( <main data-brand="brand-2" data-theme="light"> <FfButton type="submit">Save changes</FfButton> </main> ); }
PropTypeDescription
labelstringFallback label when no slotted content is provided.
disabledbooleanDisables pointer and keyboard interaction.
fullWidthbooleanExpands the host without coupling layout to a specific brand.
type'button' | 'submit' | 'reset'Mirrors the native button type attribute.
Primitive WCAG AA target

FfDropdown

Accessible dropdown primitive with keyboard support, active option management, selection events, and shared listbox semantics owned by the Stencil runtime.

const releaseTargetOptions = [ { value: 'pilot', label: 'Pilot release', description: 'Small audience rollout.' }, { value: 'full', label: 'Production rollout', description: 'Full audience launch.' } ]; <FfDropdown label="Release target" placeholder="Choose a release target" options={releaseTargetOptions} onFfValueChange={(customEvent) => { console.log(customEvent.detail.value); }} />
Prop or eventTypeDescription
labelstringVisible label and accessible name for the trigger.
optionsFfDropdownOption[]Full option list rendered in the listbox panel.
valuestring | undefinedCurrently selected value, optionally controlled by the consumer app.
ffValueChangeCustomEventFires when a new option is selected.
ffOpenChangeCustomEventFires when the panel opens or closes.

Theming and brand activation

The wrapper never decides how the component should look. The React app shell imports the token contract and a brand pack, then activates the visual context with data-brand and data-theme.

<main data-brand="brand-2" data-theme={activeTheme}> <FfButton>Launch selected brand</FfButton> <FfDropdown ... /> </main>

Supported override paths

App shell data-brand and data-theme context.
Documented CSS custom properties from the shared token contract.
Host classes passed directly from the consuming React app.

Accessibility expectations

Accessibility behavior belongs to the shared primitive layer, but the React wrapper is responsible for preserving that contract in a React-friendly consumption model.

AreaCurrent expectation
Keyboard supportButtons keep native button semantics. Dropdown supports Arrow keys, Enter, Escape, and Tab.
ARIA modelDropdown uses a listbox pattern with the label, expanded state, and selected state surfaced by the primitive.
Theme qualityOfficial brand packs are expected to meet WCAG AA contrast targets before release.

Release and maintenance model

The wrapper is intended to ship as part of a centrally versioned platform. Product teams should consume it as a package. Shadcn-style copy ownership may exist only as an escape hatch for exceptional product requirements, not as the default path.

Recommended enterprise policy

Default: consume the official wrapper package and official brand styles.
Exception: copy ownership only with explicit ownership and upgrade responsibility.
Never fork the official corporate brand pack as the primary delivery model.
