Angular app · external CSS brand pack

Angular consumes only wrapper logic, then Brand 1 arrives from CSS.

This application imports @fuggetlenfe/angular-wrapper, plus @fuggetlenfe/tokens/contract.css and @fuggetlenfe/brand-styles/brand-1-light.css together with @fuggetlenfe/brand-styles/brand-1-dark.css. The component library itself stays visually neutral until the external stylesheet is present.

Consumer imports

import { FfButton } from '@fuggetlenfe/angular-wrapper'
@import '@fuggetlenfe/tokens/contract.css';
@import '@fuggetlenfe/brand-styles/brand-1-light.css';
@import '@fuggetlenfe/brand-styles/brand-1-dark.css';

<main data-brand="brand-1" [attr.data-theme]="theme()">
  <ff-button></ff-button>
</main>

Theme behavior

[attr.data-theme]=\"theme()\"
data-theme=\"{{ theme() }}\"

What proves the separation

  • The Angular wrapper provides framework integration only.
  • The imported CSS library gives the component its brand and surface tokens.
  • The same wrapper logic swaps between light and dark with CSS alone.
  • Host classes and per-instance CSS properties still tailor the component locally.
