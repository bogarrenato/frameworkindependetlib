Angular app · custom brand CSS lib

The same Angular wrapper gets a custom identity from a separate stylesheet.

The component and wrapper stay logic-focused. This application applies a completely separate visual system by importing the custom light and dark brand packs next to the neutral contract CSS.

Consumer imports

import { FfButton } from '@fuggetlenfe/angular-wrapper'
@import '@fuggetlenfe/tokens/contract.css';
@import '@fuggetlenfe/brand-styles/custom-brand-light.css';
@import '@fuggetlenfe/brand-styles/custom-brand-dark.css';
<main data-brand="custom-brand" [attr.data-theme]="theme()">
  <ff-button></ff-button>
</main>

Theme behavior

The shell flips between light and dark.
The button keeps the same logic API in both modes.
Only the imported custom CSS pack changes the look.

Why this matters

The Angular wrapper stays reusable across multiple products.
The brand arrives from a different CSS package, not from the component code.
The same wrapper logic swaps between light and dark with CSS alone.
Host classes and custom CSS properties still allow local ownership and tuning.
